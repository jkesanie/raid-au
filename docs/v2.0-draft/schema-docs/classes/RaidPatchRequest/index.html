    ContributorPosition

    RaidPatchRequest.Contributor.ContributorPosition

    Description:

    a metadata schema sub-block describing a contributor’s administrative position on a project or activity

    Requirement: mandatory

    Cardinality: 1..*

    constraints: contributors must have one and only one position at any given time (contributors may also be flagged as a ‘leader’ or ‘contact’ separately).

    notes: This property represents a contributor’s administrative position on a project (such as their position on a grant application); use contributor.role to define for scientific or scholarly contribution.

    RaidPatchRequest.Contributor.ContributorPosition.startDate

    Description:

    date the contributor began position associated with the project or activity

    Requirement: mandatory

    Cardinality: 1

    notes: ISO 8601 standard date, Only the year is required; month and day are optional (but recommended when available).

    RaidPatchRequest.Contributor.ContributorPosition.endDate

    Description:

    date the contributor terminated position associated with the project or activity

    Requirement: optional

    Cardinality: 0..1

    notes: ISO 8601 standard date. Only the year is required; month and day are optional (but recommended when available). Recommended if an organisation has changed roles or terminated its relationship with a project

    RaidPatchRequest.Contributor.ContributorPosition.id
